# RWB WildHack

Хакатон по машинному обучению от Wildberries & Russ на платформе [WB Space](https://wbspace.wb.ru/wildhack). Обе задачи — прогнозирование объёмов отгрузок по логистическим маршрутам складской сети на основе данных о прохождении товаров через этапы обработки.

## Метрика

В обоих треках качество оценивается одинаково — суммой WAPE и абсолютного относительного смещения (чем меньше, тем лучше):

$$\text{score} = \text{WAPE} + |\text{Relative Bias}|$$

$$\text{WAPE} = \frac{\sum |y_i - \hat{y}_i|}{\sum y_i}, \qquad \text{Relative Bias} = \left|\frac{\sum \hat{y}_i}{\sum y_i} - 1\right|$$

WAPE штрафует за абсолютную ошибку прогноза, Relative Bias — за систематический сдвиг суммарного объёма (переоценку или недооценку в целом). Минимизировать нужно обе составляющие одновременно.

**Формат сабмита:** `.csv` с двумя колонками — `id` (идентификатор строки из теста) и `y_pred` (предсказанное значение целевой переменной).

## Отгрузки без простоев (Solo трек)

**Место: 125 / 678 (топ-20%)** · [Решение](https://github.com/mmmaximov/mmkaggle/blob/main/RWB%20Wildhack/WB_SOLO.ipynb)

Прогноз объёма товаров, отгруженных по маршруту за последний час.

**Целевая переменная:** `target_1h` — объём товаров, отгруженных по маршруту за последний час.

**Признаки (тренировочный набор):**

| Колонка | Описание |
|---|---|
| `route_id` | уникальный идентификатор маршрута |
| `timestamp` | момент времени |
| `status_1`, `status_2`, `status_3` | кол-во товаров по маршруту, прошедших конкретные этапы обработки на **текущем** складе за последние 30 минут |
| `status_4`, `status_5`, `status_6` | кол-во товаров, направляющихся на текущий склад, прошедших этапы обработки на **предыдущем** складе за последние 30 минут |
| `target_1h` | целевая переменная |

Тестовый набор не содержит статусов и целевой переменной — нужно предсказать `target_1h` для каждой временной точки.

## Автоматизация логистики склада (Team трек)

**Место: 35 / 443 (топ-10%)** · [Решение](https://github.com/mmmaximov/mmkaggle/blob/main/RWB%20Wildhack/WB_TEAM.ipynb)

**Целевая переменная:** `target_2h` — количество ёмкостей, отгруженных по маршруту за последние два часа.

**Признаки (тренировочный набор):**

| Колонка | Описание |
|---|---|
| `route_id` | уникальный идентификатор маршрута |
| `office_from_id` | склад-источник маршрута (привязка маршрутов к складам фиксирована) |
| `timestamp` | момент времени |
| `status_1` … `status_8` | кол-во товаров по маршруту, прошедших конкретные этапы обработки на текущем складе за последние 30 минут |
| `target_2h` | целевая переменная |

Тестовый набор не содержит статусов и целевой переменной — нужно предсказать `target_2h` для каждой временной точки.
